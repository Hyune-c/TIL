## # Cookie vs Session  
state less 인 HTTP 의 문제점을 보완하기 위한 방식이지만,   
두 방식 모두 보안의 위험이 존재하기에 토큰을 사용하는 OAuth, JWT 방식이 나왔습니다.  

|          | Cookie            | Session       |
| -------- | ----------------- | ------------- |
| Stored   | Browser in Client | Server        |
| Security | 변조의 위험성     | 탈취의 위험성 |
|          |                   |               |
